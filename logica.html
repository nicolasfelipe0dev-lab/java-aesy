Lógica por trás dos comandos de Java

Abra a caixa-preta do Java. Entenda a engrenagem oculta por trás de cada comando, domine os pilares da programação orientada a objetos e aprenda a pensar como os grandes programadores pensam.

if / else

Avalia uma expressão booleana. Se for verdadeira, executa o primeiro bloco; se for falsa, desvia o fluxo para o bloco alternativo.

switch / case / default

Avalia o valor de uma única variável e salta diretamente para o bloco correspondente. Se nenhum valor coincidir, executa o bloco de segurança default.

break

Interrompe abruptamente a execução do laço de repetição ou bloco condicional atual, transferindo o controle para a próxima linha fora desse bloco.

continue

Interrompe a iteração atual de um laço, ignorando o código restante abaixo dele, e salta diretamente para o teste da próxima repetição.

while

Avalia uma condição antes de entrar no bloco. Se for verdadeira, executa o código e repete o teste. Só encerra quando a condição se tornar falsa.

do / while

Executa o bloco de código primeiro e só depois testa a condição. Garante que o código seja rodado pelo menos uma vez, mesmo que a condição seja falsa.

for

Inicializa uma variável de controle, testa uma condição de parada e executa um passo de incremento a cada rodada. Ideal para repetições conhecidas.

for-each

Sequencia automaticamente cada elemento de uma coleção ou array, do início ao fim, extraindo o valor atual para uma variável temporária.

class

Define la estrutura, os atributos e os comportamentos de um tipo de dado personalizado. Funciona como o projeto ou molde para criar objetos.

new

Solicita memória ao sistema operacional para criar uma instância real de uma classe e ativa o construtor para inicializar o objeto.

this

Funciona como uma referência autoreferencial, apontando para a própria instância do objeto que está executando o código naquele momento.

super

Estabelece uma ponte direta com a classe pai imediata, permitindo acessar seus construtores, métodos ou atributos que foram herdados.

extends

Estabelece uma relação de herança entre duas classes, fazendo com que a classe filha adquira automaticamente as características e comportamentos da classe pai.

implements

Assina um compromisso onde a classe se obriga a fornecer uma implementação real para todos os métodos abstratos definidos por uma interface.

interface

Define um contrato de comportamento puro, especificando o que uma classe deve fazer, mas deixando totalmente em aberto o como ela fará.

abstract

Cria modelos incompletos. Se aplicada a uma classe, impede sua instanciação; se aplicada a um método, obriga as subclasses a criarem a lógica.

public

Remove qualquer barreira de visibilidade, permitindo que o elemento seja acessado por qualquer outra classe em qualquer pacote do projeto.

private

Restringe o acesso ao nível mais rígido, tornando o elemento visível e modificável exclusivamente de dentro da própria classe onde foi declarado.

protected

Permite o acesso ao elemento apenas por classes que estejam no mesmo pacote ou por subclasses, mesmo que estejam em pacotes diferentes.

static

Desvincula o elemento de instâncias individuais. O atributo ou método passa a pertencer à classe em si e compartilha a mesma memória para todos.

final

Define uma restrição de imutabilidade. Impede a alteração do valor em variáveis, a sobrescrita em métodos ou a herança em classes.

try / catch

O try monitora falhas imprevisíveis. Se uma falha acontece, o fluxo normal é interrompido e o catch assume para tratar o problema sem quebrar o sistema.

finally

Garante a execução de um bloco de código de limpeza, independentemente de o código no try ter rodado com sucesso ou ter disparado um erro.

throw

Dispara deliberadamente uma exceção (erro controlado) no fluxo do programa, interrompendo a execução atual para sinalizar uma anomalia.

throws

Declara explicitamente na assinatura de um método que ele pode produzir erros, transferindo a responsabilidade de tratamento para quem invocá-lo.

void

Sinaliza que o método realiza um processamento ou ação, mas não devolve nenhum tipo de dado ou resposta ao final de sua execução.

return

Encerra a execução do método atual imediatamente e envia um valor ou objeto de volta para o ponto exato onde o método foi chamado.
